- 현재 프로토타입 프로젝트이기 때문에 이메일 인증이나 보안기능 등을 사용하지 않아야 함


2) 데이터베이스 설계



** Users (사용자) **
- id (UUID, Primary Key)
- username (String, Unique, Not Null) - 아이디
- password (String, Not Null) - 비밀번호 (해시 처리)
- nickname (String, Not Null) - 닉네임
- profile_image (String, Nullable) - 프로필 이미지 URL
- bio (Text, Nullable) - 자기소개
- website (String, Nullable) - 웹사이트
- is_private (Boolean, Default: false) - 비공개 계정 여부
- show_activity_status (Boolean, Default: true) - 활동 상태 표시
- created_at (Timestamp)
- updated_at (Timestamp)

** Posts (게시물) **
- id (UUID, Primary Key)
- user_id (UUID, Foreign Key → Users.id)
- caption (Text, Nullable) - 게시물 내용
- location (String, Nullable) - 위치/장소
- created_at (Timestamp)
- updated_at (Timestamp)

** Post_Images (게시물 이미지) **
- id (UUID, Primary Key)
- post_id (UUID, Foreign Key → Posts.id)
- image_url (String, Not Null) - 이미지 URL
- order_index (Integer, Not Null) - 이미지 순서
- created_at (Timestamp)

** Likes (좋아요) **
- id (UUID, Primary Key)
- user_id (UUID, Foreign Key → Users.id)
- post_id (UUID, Foreign Key → Posts.id)
- created_at (Timestamp)
- Unique Constraint: (user_id, post_id)

** Comments (댓글) **
- id (UUID, Primary Key)
- user_id (UUID, Foreign Key → Users.id)
- post_id (UUID, Foreign Key → Posts.id)
- parent_comment_id (UUID, Foreign Key → Comments.id, Nullable) - 대댓글용
- content (Text, Not Null) - 댓글 내용
- created_at (Timestamp)
- updated_at (Timestamp)

** Comment_Likes (댓글 좋아요) **
- id (UUID, Primary Key)
- user_id (UUID, Foreign Key → Users.id)
- comment_id (UUID, Foreign Key → Comments.id)
- created_at (Timestamp)
- Unique Constraint: (user_id, comment_id)

** Follows (팔로우) **
- id (UUID, Primary Key)
- follower_id (UUID, Foreign Key → Users.id) - 팔로우하는 사람
- following_id (UUID, Foreign Key → Users.id) - 팔로우받는 사람
- created_at (Timestamp)
- Unique Constraint: (follower_id, following_id)

** Message_Rooms (메시지방) **
- id (UUID, Primary Key)
- created_at (Timestamp)
- updated_at (Timestamp)

** Message_Room_Participants (메시지방 참여자) **
- id (UUID, Primary Key)
- room_id (UUID, Foreign Key → Message_Rooms.id)
- user_id (UUID, Foreign Key → Users.id)
- joined_at (Timestamp)
- Unique Constraint: (room_id, user_id)

** Messages (메시지) **
- id (UUID, Primary Key)
- room_id (UUID, Foreign Key → Message_Rooms.id)
- sender_id (UUID, Foreign Key → Users.id)
- content (Text, Nullable) - 메시지 내용
- image_url (String, Nullable) - 이미지 URL
- created_at (Timestamp)
- is_read (Boolean, Default: false) - 읽음 여부

** Notifications (알림) **
- id (UUID, Primary Key)
- user_id (UUID, Foreign Key → Users.id) - 알림을 받는 사람
- sender_id (UUID, Foreign Key → Users.id) - 알림을 발생시킨 사람
- type (Enum: 'like', 'comment', 'follow', 'mention') - 알림 타입
- post_id (UUID, Foreign Key → Posts.id, Nullable)
- comment_id (UUID, Foreign Key → Comments.id, Nullable)
- is_read (Boolean, Default: false) - 읽음 여부
- created_at (Timestamp)

** Saved_Posts (저장한 게시물) **
- id (UUID, Primary Key)
- user_id (UUID, Foreign Key → Users.id)
- post_id (UUID, Foreign Key → Posts.id)
- created_at (Timestamp)
- Unique Constraint: (user_id, post_id)

** Blocked_Users (차단한 사용자) **
- id (UUID, Primary Key)
- blocker_id (UUID, Foreign Key → Users.id) - 차단한 사람
- blocked_id (UUID, Foreign Key → Users.id) - 차단당한 사람
- created_at (Timestamp)
- Unique Constraint: (blocker_id, blocked_id)

** 인덱스 설정 **
- Users: username (Unique), nickname
- Posts: user_id, created_at
- Post_Images: post_id, order_index
- Likes: user_id, post_id, created_at
- Comments: post_id, user_id, parent_comment_id, created_at
- Comment_Likes: user_id, comment_id
- Follows: follower_id, following_id
- Messages: room_id, sender_id, created_at
- Notifications: user_id, is_read, created_at
- Saved_Posts: user_id, post_id

4) 깊이 관계 및 정보 아키텍처

** 페이지 네비게이션 플로우 (깊이 구조) **

Level 0 (인증 전)
└─ 로그인 페이지
   ├─ 회원가입 페이지
   └─ 비밀번호 찾기 페이지

Level 1 (메인 레이아웃 - 하단바 네비게이션)
├─ 메인 페이지 (피드/타임라인) [하단바: 홈]
│  ├─ Level 2: 게시물 상세 페이지
│  │  └─ Level 3: 댓글 작성자 프로필 페이지
│  ├─ Level 2: 게시물 작성자 프로필 페이지
│  │  ├─ Level 3: 팔로워/팔로잉 목록 페이지
│  │  └─ Level 3: 게시물 상세 페이지
│  ├─ Level 2: 알림 페이지 (상단바 알림 아이콘)
│  │  └─ Level 3: 관련 게시물 상세 페이지
│  └─ Level 2: 메시지 페이지 (상단바 메시지 아이콘)
│     └─ Level 3: 대화 상세 페이지
│        └─ Level 4: 상대방 프로필 페이지
│
├─ 검색 페이지 [하단바: 검색]
│  ├─ Level 2: 검색 결과 사용자 프로필 페이지
│  │  ├─ Level 3: 팔로워/팔로잉 목록 페이지
│  │  └─ Level 3: 게시물 상세 페이지
│  └─ Level 2: 해시태그 검색 결과 페이지 (선택사항)
│
├─ 게시물 업로드 페이지 [하단바: +]
│  ├─ Level 2: 이미지 선택 페이지
│  ├─ Level 2: 이미지 편집 페이지 (필터, 크롭)
│  └─ Level 2: 위치/사람 태그 선택 페이지
│
└─ 프로필 페이지 (내 프로필) [하단바: 프로필]
   ├─ Level 2: 프로필 편집 페이지
   ├─ Level 2: 설정 페이지
   │  ├─ Level 3: 비밀번호 변경 페이지
   │  ├─ Level 3: 계정 설정 페이지
   │  ├─ Level 3: 알림 설정 페이지
   │  ├─ Level 3: 개인정보 설정 페이지
   │  │  ├─ Level 4: 차단 사용자 관리 페이지
   │  │  └─ Level 4: 숨긴 계정 관리 페이지
   │  └─ Level 3: 계정 삭제 확인 페이지
   ├─ Level 2: 팔로워/팔로잉 목록 페이지
   │  └─ Level 3: 팔로워/팔로잉 프로필 페이지ㄴ
   ├─ Level 2: 저장한 게시물 페이지
   │  └─ Level 3: 게시물 상세 페이지
   └─ Level 2: 내 게시물 상세 페이지
      └─ Level 3: 게시물 편집/삭제 모달

** 페이지 깊이별 특징 **
- Level 0: 인증 전 페이지 (로그인, 회원가입)
- Level 1: 메인 네비게이션 (하단바로 접근, 앱의 핵심 페이지)
- Level 2: Level 1에서 1클릭으로 접근하는 세부 페이지
- Level 3: Level 2에서 1클릭 더 들어간 페이지
- Level 4: 가장 깊은 페이지 (최대 4단계로 제한하여 사용자 경험 보호)

** 네비게이션 규칙 **
1. 뒤로가기: Level 2 이상의 모든 페이지는 상단바에 뒤로가기 버튼 필수
2. 홈으로: 하단바의 홈 아이콘 클릭 시 Level 1 메인 페이지로 즉시 이동
3. 탭 전환: 하단바의 다른 탭 클릭 시 해당 Level 1 페이지로 즉시 이동
4. 깊이 제한: 최대 4단계까지만 허용 (사용자가 길을 잃지 않도록)

** 정보 아키텍처 (IA) 요약 **

1단계 (Primary Navigation - 하단바)
├─ 홈 (피드) - 게시물 타임라인, 앱의 메인
├─ 검색 - 유저 검색, 추천 사용자
├─ 업로드 - 새 게시물 작성
└─ 프로필 - 내 프로필, 게시물 그리드

2단계 (Secondary Navigation)
├─ 홈에서
│  ├─ 게시물 상세 (클릭)
│  ├─ 프로필 (닉네임 클릭)
│  ├─ 알림 (상단바 아이콘)
│  └─ 메시지 (상단바 아이콘)
├─ 검색에서
│  └─ 검색 결과 프로필
└─ 프로필에서
   ├─ 프로필 편집
   ├─ 설정
   ├─ 팔로우 목록
   └─ 저장한 게시물

3단계 (Tertiary Navigation)
├─ 게시물 상세에서
│  └─ 댓글 작성자 프로필
├─ 설정에서
│  ├─ 비밀번호 변경
│  ├─ 계정 설정
│  └─ 개인정보 설정
└─ 팔로우 목록에서
   └─ 팔로워/팔로잉 프로필

4단계 (최대 깊이)
└─ 개인정보 설정에서
   ├─ 차단 사용자 관리
   └─ 숨긴 계정 관리

** 네비게이션 우선순위 **
High Priority (자주 사용):
- 홈 → 게시물 상세
- 홈 → 프로필
- 검색 → 프로필
- 프로필 → 게시물 상세

Medium Priority:
- 홈 → 알림
- 홈 → 메시지
- 프로필 → 팔로우 목록
- 프로필 → 프로필 편집

Low Priority:
- 프로필 → 설정
- 설정 → 세부 설정 페이지

** 사용자 흐름 예시 **

신규 사용자:
회원가입 → 로그인 → 홈(빈 피드) → 검색 → 유저 찾기 → 팔로우 → 홈(피드 채워짐)

게시물 업로드:
홈 → 업로드 버튼 → 이미지 선택 → 편집 → 캡션 작성 → 공유 → 홈(내 게시물 표시)

타인 프로필 방문:
홈 → 게시물 클릭 → 닉네임 클릭 → 프로필 → 팔로우 → 게시물 확인

메시지 전송:
홈 → 메시지 아이콘 → 새 메시지 → 유저 선택 → 메시지 작성 → 전송
